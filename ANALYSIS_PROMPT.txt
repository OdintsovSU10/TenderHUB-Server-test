ЗАДАНИЕ ДЛЯ АНАЛИЗА ПРОЕКТА

Проанализируй прикреплённый проект TenderHUB - портал управления строительными тендерами.

ТЕКУЩАЯ АРХИТЕКТУРА:
- Frontend: React 18 + TypeScript + Vite
- Backend: Supabase (BaaS) - PostgreSQL + Auth + RLS
- State Management: TanStack Query + React Context
- UI: Ant Design 5
- Паттерны: Clean Architecture (Domain/Ports/Adapters)

ЦЕЛЬ АНАЛИЗА:
Оцени возможность и целесообразность перехода на полноценную клиент-серверную архитектуру с выделенным бэкендом.

ВОПРОСЫ ДЛЯ АНАЛИЗА:

1. ТЕКУЩЕЕ СОСТОЯНИЕ:
   - Какие бизнес-логики сейчас находятся на клиенте и должны быть на сервере?
   - Какие есть потенциальные проблемы безопасности при текущем подходе?
   - Насколько хорошо реализован паттерн Repository для абстракции доступа к данным?

2. ВАРИАНТЫ МИГРАЦИИ:
   - Supabase Edge Functions (Deno) - минимальные изменения
   - Node.js/Express/NestJS backend - классический подход
   - Go/Rust backend - высокопроизводительный вариант

   Для каждого варианта оцени:
   - Сложность миграции
   - Преимущества и недостатки
   - Примерный объём работ

3. ПЛАН МИГРАЦИИ:
   - Какие модули мигрировать в первую очередь?
   - Как обеспечить бесшовный переход без простоя?
   - Какие API endpoints нужно создать?

4. РЕКОМЕНДАЦИИ:
   - Стоит ли вообще уходить от Supabase?
   - Какой стек порекомендуешь для бэкенда?
   - Какие архитектурные паттерны применить?

СТРУКТУРА ПРОЕКТА:
- src/core/ - доменный слой (entities, ports, services)
- src/client/ - адаптеры (repositories, providers)
- src/pages/ - страницы приложения (20+)
- src/components/ - UI компоненты
- src/lib/supabase/ - клиент Supabase
- supabase/schemas/prod.sql - схема БД (26 бизнес-таблиц)
